# Work Log - 2026-03-10

## Summary
- Implemented Firebase anonymous-auth foundation in Android + backend token verification scaffold.
- Implemented backend-enforced monthly scan quota policy with atomic counting.
- Added idempotency support (`request_id`) to avoid double-charging scan usage on retries.
- Documented new env vars and quick validation steps.

## Detailed Work Completed
1. Firebase auth wiring in Android
- Added Google Services plugin to Gradle setup.
- Added Firebase auth dependency compatible with current Kotlin/Gradle stack.
- Added `FirebaseAuthManager` to sign in anonymously and fetch bearer token.
- App now attempts anonymous sign-in on startup (`MenuLensApp`).
- Scan requests now include `Authorization: Bearer <firebase_id_token>` when available.

2. Backend Firebase identity support
- Added `firebase-admin` dependency.
- Added optional Firebase token verification in `/v1/scan_menu`.
- Added rollout guard:
  - `ENABLE_FIREBASE_AUTH=false` => token optional (fallback to device identity).
  - `ENABLE_FIREBASE_AUTH=true` => token required and verified.

3. Monthly quota implementation (`free=10/month`, `pro=250/month`)
- Added new module `backend/app/usage.py` with `UsageStore`.
- Store features:
  - Atomic check-and-increment per `(subject_key, period_ym)` using SQLite transaction.
  - Plan-aware quotas (`free_quota`, `pro_quota`).
  - Idempotency table keyed by `(subject_key, request_id)`.
- `scan_menu` now:
  - Resolves subject key (`uid:<firebase_uid>` or `device:<device_id>` fallback).
  - Consumes usage before expensive OCR/Gemini pipeline.
  - Returns `402` with `code=scan_quota_exceeded` when monthly cap reached.

4. Idempotency request field
- Added `request_id` multipart field in Android API interface + repository.
- Android now generates per-scan UUID request ID.
- Backend consumes the same `request_id` for duplicate suppression.

5. Diagnostics and docs
- Added usage fields in `pipeline_diagnostics`:
  - `usage_period_ym`
  - `usage_plan`
  - `usage_scans_used`
  - `usage_scans_quota`
  - `usage_scans_remaining`
  - `usage_duplicate_request`
- Updated backend README env and testing notes.

6. Validation performed
- Android unit tests: `./gradlew testDebugUnitTest` passed (JDK 17).
- Backend syntax checks: `python -m compileall app/main.py app/usage.py` passed.
- Local usage smoke test verified:
  - first request counts
  - same `request_id` is deduped
  - quota limit triggers block
