## Detailed Work Completed
1. Prompt versioning foundation
- Added versioned prompt files under `backend/app/prompts/`:
  - `menu_parse_v1.txt`
  - `ocr_normalize_v1.txt`
- Added `backend/app/prompts/registry.py` to resolve and render prompts by version.
- Added env-selectable prompt versions:
  - `MENU_PARSE_PROMPT_VERSION`
  - `OCR_NORMALIZE_PROMPT_VERSION`
- Updated `backend/app/main.py` to load prompts through the registry instead of embedding prompt strings inline.
- Added prompt version IDs to live `pipeline_diagnostics`.

2. Offline eval package
- Added `backend/evals/` with:
  - dataset schema
  - real fixture labels
  - eval runner
  - scoring module
  - eval README
  - results directory placeholder
- Added 8 menu fixture images under `backend/evals/fixtures/`.
- Added `backend/evals/dataset/examples.json` with expected Japanese item names, expected item counts, difficulty tags, notes, and optional English titles.

3. Eval runner outputs
- Added `python -m evals.run_evals` workflow from the `backend/` directory.
- Eval runner now writes full timestamped JSON reports under `backend/evals/results/`.
- Eval runner also appends a summary row to `backend/evals/results/eval_runs.csv`.
- CSV ledger includes:
  - run timestamp
  - report path
  - dataset path
  - target language
  - model
  - OCR pipeline mode
  - prompt versions
  - failure count
  - parse success rate
  - item recall
  - precision proxy
  - hallucinated item rate
  - coverage ratio
  - mean and p95 latency

4. Scoring improvements
- Replaced exact-only matching with one-to-one fuzzy item matching.
- Added normalization for whitespace and punctuation.
- Added common Japanese menu spelling canonicalization for kana/kanji variants, such as sashimi spelling variants.
- Added per-example scoring diagnostics:
  - `matches`
  - `missed_expected_jp_names`
  - `extra_predicted_jp_names`
- Added regression checks for important edge cases:
  - spacing differences should not be penalized
  - extra descriptors can still match an expected item name
  - overly generic predictions should not match more specific item names
  - missing specific ingredients should not pass as a match

  - item precision proxy: `0.7704`
